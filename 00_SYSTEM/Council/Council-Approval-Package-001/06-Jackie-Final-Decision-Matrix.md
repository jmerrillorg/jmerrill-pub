## Separate Approval Block - DEC-CAPSTD-001

I approve JM1 Enterprise Capability & Pipeline Standard v1.0 for promotion from CANON-CANDIDATE to CANON. This approval does not authorize production implementation, register schema deployment, Annex A authority, or blanket J0-J8 certification.

## Separate Approval Block - DEC-CAPCAT-001

I approve the Capability Catalog schema v0.1 and 13-row seed v0.1 as the governed repository register template and current repository seed. This approval does not authorize Dataverse table creation, production migration, or automatic capability promotion.

## Separate Approval Block - DEC-PIPEREG-001

I approve the Pipeline Register schema v0.1 and corrected J0-J8 seed v0.1 as the current repository record. J0-J8 is the designated first JM1 enterprise reference implementation, certified stage by stage through the Pipeline Register. This approval does not blanket-certify all stages or authorize production implementation.

## Separate Approval Block - DEC-AGTREG-001

I approve the Agent Registry v2 schema package as the future governed registry model. This approval permits repository-level schema adoption and separately approved migration planning only. It does not authorize live migration, agent activation, A3 promotion, runtime deployment, or commissioning.

## Separate Approval Block - DEC-ANNEXA-001

I approve Annex A as authoritative with the named exceptions listed in Council-Approval-Package-001. This approval does not authorize implementation, cross-pillar unified views, autonomy expansion, runtime deployment, schema changes, or agent commissioning. Each exception remains subject to follow-up evidence review.
