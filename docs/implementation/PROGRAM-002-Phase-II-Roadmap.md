# PROGRAM-002 Phase II Roadmap

Program: PROGRAM-002 - Autonomous Publishing Production Pipeline

Status: Phase II planning baseline

Source baseline: PROGRAM-002 Phase I certified state

## Purpose

Phase II moves PROGRAM-002 beyond Active Publishing workspace commissioning into controlled release, royalty, post-publication, catalog adoption, and returning-author operations.

Phase II must not reinterpret Phase I. It begins from the certified baseline:

- author relationship active
- agreement signed
- payment confirmed
- workspace unlocked
- production pipeline visible
- distribution held at Commissioning Hold

## Major Phase II Milestones

### 1. Distribution Release

Goal: Move a title from Commissioning Hold to authorized distribution release only after title-specific approval.

Required outcomes:

- release approval model
- final metadata confirmation
- final files confirmation
- channel readiness
- no accidental retailer/Ingram/public submission before approval
- release execution evidence

Not included:

- public launch campaign
- royalty payments
- Business Central posting

### 2. Royalty Automation

Goal: Prepare royalty statement and payment readiness after release/reportable activity exists.

Required outcomes:

- royalty source-of-truth confirmation
- statement readiness
- payout eligibility rules
- author-facing royalty explanation
- payment gating
- no unauthorized author payment

Not included:

- tax advice
- unaudited payout automation
- QBO logic

### 3. Post-Publication Author Success

Goal: Govern the author relationship after release.

Required outcomes:

- support cadence
- catalog health review
- post-release issue handling
- future title opportunity path
- author success workspace view
- annual/review handoff awareness without starting unrelated milestones

### 4. Catalog Adoption / OP-000

Goal: Certify legacy, active, and published JMP titles into PROGRAM-002 without forcing lifecycle restart.

Required outcomes:

- catalog/title inventory
- current-state certification
- workspace linking
- evidence import
- relationship state
- correct resume gate
- no duplicate contracts, workspaces, royalties, or production steps

### 5. Legacy Author Workspace

Goal: Provide safe workspace access for authors whose titles existed before PROGRAM-002.

Required outcomes:

- author identity matching
- title linking
- historical evidence display
- active/inactive relationship state
- sensitive material boundaries

### 6. Returning Author Experience

Goal: Support additional titles for existing authors without creating duplicate author portals.

Required outcomes:

- parent author / child title behavior
- title switcher or title list
- onboarding update rules
- package/recommendation path for new title
- agreement/payment path per title

## Phase II Guardrails

Do not proceed without title-specific authorization when an action would:

- submit to retailers
- submit to Ingram
- create preorder/public availability
- launch public marketing
- generate royalties
- send author payment
- post to Business Central
- alter signed agreement evidence
- duplicate workspace/contract/opportunity records

## Recommended Sequence

1. Distribution Release
2. Post-Publication Author Success visibility
3. Royalty Automation readiness
4. OP-000 Catalog Adoption
5. Legacy Author Workspace
6. Returning Author Experience

Distribution Release should come first because The Intentional Leader is currently held at the release boundary.

## Phase II Entry Criteria

- Phase I closeout docs merged.
- Phase I certified baseline accepted by Jackie.
- Current commissioning title remains in Commissioning Hold until release approval.
- No unresolved production-risk blocker exists in active workspace, agreement, payment, or author status.
